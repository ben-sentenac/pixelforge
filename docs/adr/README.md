# Architecture Decision Records

Ce dossier contient les décisions d'architecture de PixelForge.

Un ADR explique :

* le problème rencontré ;
* les options possibles ;
* la décision prise ;
* les conséquences de cette décision.

Les ADR ne remplacent pas `docs/architecture.md`.

Ils servent à conserver l'historique des choix importants du projet.

---

# Format

Chaque ADR suit ce modèle :

```text
# ADR-XXXX — Title

Status: Accepted

Date: YYYY-MM-DD

## Context

## Decision

## Consequences
```
