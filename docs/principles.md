# PixelForge Principles

PixelForge is a learning project and a modular Bash prompt engine.

## Core principles

1. Understand before abstracting.
2. Data before rendering.
3. One responsibility per layer.
4. Providers collect data.
5. Widgets render information.
6. Layouts compose widgets.
7. Skins define visual identity.
8. No hidden side effects.
9. Small Bash functions.
10. No unnecessary dependencies.
11. Explicit APIs over magic.
12. Architecture over feature count.

## Abstraction rule

A new abstraction is introduced only when it is justified by at least two real use cases.
